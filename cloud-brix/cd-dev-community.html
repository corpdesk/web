CORPDESK Development

  • Editor
  • Jan 8, 2021
  • 12 Comments

Below is step by step process of starting COPRDESK Development

Pre-requistes

To get started you need some proficiency in Typescript and Angular Development. Another advantage is basic knowledge on webpack module federation.

Getting Started

  • Clone RemoteApp template

  • Set the app name

  • Set the port

  • Create and initialize your repository

  • Add some module with routing

  • Add some module component and some test logics

  • Update routing

  • Test access to routing via the local ports

  • Clone ShellApp template

  • Set up module federation for ShellApp to consume the RemoteApp

  • Set up the ShellApp routing for the RemoteApp navigation

Going Further

  • The template you started with already has pre-installed libraries that have rich features. The library @corpdesk/core is used to access CORPDESK back end. It also facilitate harmonization of developer codes from one application to another.

    Read about more features in the link below:

    CORPDESK Libraries

  • You can take advantage of already done base applications like cd-user, cd-comm and pms. This allows you to focus on the logics of your app logics.

    Read about more features in the link below:

    Core Applications

  • When you access your application in the ShellApp, all navigation function and guards are already done.

    Read about more features in the link below:

    ShellApp

  • To understand how different components are structured in CORPDESK intrastructure, you can visit the link below.

    CORPDESK Architecture
