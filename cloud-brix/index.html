cloud-brix®

  • Editor
  • July 8, 2021
  • 12 Comments

cloud-brix® is a visionary technology that allows users to set up their own cloud ecosystem without having resort to popular services by AWS, Google or Microsoft (see pros and cons of popular cloud services). It is developed with deliberate response to issues raised by very many cloud consumers who have experienced challenges with the regular cloud services that are available online.

The value of adopting cloud-brix® system is particularly significant when you consider long term cost analysis and administrative autonomy, including other factors that are discussed later in this article (advantages of on premise cloud system). Of course with cloud-brix® , you get to have all the advantages that goes with adopting cloud technology.

Architecture

All application and services nodes are preceeded by a load balancer with virtual IP as shown above:

Layer 1: Array of physical machines. The physical network is independent of the upper layers consisting of the Linux cluster and the application layer.

Layer 2: Linux containers are the actual host for the distributed applications. The containers are easily disposable and fast and recreate. Creations and setting up of containers is automated (managed).

Layer 3: Application layer receives requests to various services via a virtual IP floating in between HAProxy servers. This allows services to be scalable and disposable without interrupting services.

Easy does it!

Setting up a cloud system is complex. cloud-brix® circumvent this challenge by availing hardware and accessories that are already configured and ready to run.

CORPDESK as cloud-brix® Engine

cloud-brix ecosystem is powered by CORPDESK.

CORPDESK is a modular distributed application. cloud-brix® relies on it for graphic users interface, database, cache and push services. Ref: CORPDESK

cloud-brix® Services

cloud-brix® enable users to easily deploy and manage applications. One of the most common and popular system for cloud platform is kubernetes. User can target either kubernetes base or otherwise. Where user prefers, kubernetes based cloud application, all the standard features and services are supported. Equally a developer may prefer other methods but needs multiple machines to run a given application, the cloud-brix® environment is still flexible to accommodate custom setup.

Deploying An Application project:

  • Select technology (Angular, React, Node.js, Python)
  • Select Database technology (MySQL, PostgreSQL).
  • Propagate application for load balancing and fail overs
  • set up push services
  • set up cache services
